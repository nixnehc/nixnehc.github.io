>LaTeX< The Philosophy of Latex

✍️ 2021-07-01| ⏳ 4 minutes| 🧐 ChenXin| 💾 ##toolkit

本文主要包括一些关于 LaTeX 的编写技巧和习惯。 这些内容大都来自网络，觉得有道理便收集到此处。不定期更新。

# 公式部分

  1. 行间公式用 \[ \] 并分作三行，便于日后检查。
    \[ \] 其实是 equation* 环境的缩写，因此在其中不能用 \\ 命令换行。

  2. 行间公式也是句子的一部分，不过被强行短行，因此不要忽略标点符号。

  3. 数学粗体用 \bm{}，当追求不同显示效果时可以用 \mathbf{}。

  4. : 是一个二元关系符号，以下是一些常见的误用

  • 定义符号用 \coloneqq 而不是 :=，后者的冒号会相对等号下移。同理在 BNF 规则中用 \Coloneqq。
  • 定义函数时用 f\colon A \to B 而不要写成 f : A \to B.

  1. 尖括号用 \langle \rangle 而不是直接用大于、小于符号。

  2. \to 和 \rightarrow 是同一个符号，为了简便用前者即可。尤其是在写蕴涵式的时候。

  3. \mid 是一个二元关系，而 | 是一个 math ordinal，因此 BNF 中的分隔线要用前者，如 p \mid \neg \varphi \mid \Diamond \varphi.

# Misc.

  • 数学方面的学术写作参考 Writing Scientific Documents Using LaTeX

  • 大型文档用 \input 或 \include 分出章节，模块化管理易于调试。

  • 大型项目中，源码的单个 .tex 文件最好不要超过 1000 行，方便校对。

  • 在不希望分割的词组间用 ~ 连接，如 this theorem from~\cite{article} （引文不应该出现在首行）；Blackburn~et~al~2001 （et al 不应该被分开）。

  • 单行注释用 %。多行注释可以在每行开头插入 %，也可以用如下环境（注意是 comment 而不是复数 comments）

        \begin{comment}
    这是
    一条
    多行注释
    \end{comment}

# 编辑器 TeXStudio

  • 字体，从可读性和美观上考量

    • macOS 上用 Menlo 字体
    • windows 上用 consolas 字体
    • linux 上用 dejavu san mono 字体

  • 要利用其强大的「宏」功能

  • 用 Ctrl+f、Ctrl+b、Ctrl+n、Ctrl+p 代替方向键，这几个键位虽不如 vim 中那样方便，但胜在 MacOS 上全局通用。
    记得互换 CapsLock 和 Ctrl 的键位，保护小指健康😘
